# VT Business

Локальный проект для управления строительными объектами в Бельгии.

## Подход

Сначала пишем спеки, потом переходим к реализации.

1. Сначала глобальная рамка.
2. Потом бизнес-контекст.
3. Потом функциональная карта.
4. Потом технический план.
5. Потом scope первой версии.

## Структура

```text
VT-bussines/
  AGENTS.md
  README.md
  spec/
    00-глобальная-рамка.md
    01-бизнес-контекст.md
    02-функциональная-карта.md
    03-технический-план.md
    04-scope-v1.md
```

## Главные файлы

- `spec/00-глобальная-рамка.md` - фиксирует смену рамки проекта на простой сайт.
- `spec/01-бизнес-контекст.md` - объясняет, что именно это за сайт и зачем он нужен.
- `spec/02-функциональная-карта.md` - описывает страницы, сценарии и поведение сайта.
- `spec/03-технический-план.md` - описывает, как это реализовать локально.
- `spec/04-scope-v1.md` - фиксирует, что входит и что не входит в первую версию.
