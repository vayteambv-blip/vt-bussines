# Scope первой версии

## Цель
Сделать простой локальный сайт для контроля строительных объектов в Бельгии.

## Что входит
- главная страница со списком объектов;
- карточка объекта;
- страница фирмы;
- календарь очередности;
- история изменений;
- фото и документы;
- денежный блок;
- поиск и фильтры;
- добавление и редактирование записей;
- локальный AI-черновик или комментарий.

## Что должно работать
- пользователь открывает список объектов;
- пользователь открывает карточку объекта и видит всё важное в одном месте;
- пользователь создаёт новый объект с обязательными данными;
- пользователь меняет статус, добавляет работы, расходы, фото и документы;
- пользователь видит историю изменений;
- пользователь открывает фирму и связь с объектами;
- пользователь смотрит календарь очередности;
- пользователь сохраняет локальный AI-черновик в карточку.

## Что не входит
- авторизация;
- онлайн-доступ;
- бэкенд;
- синхронизация между устройствами;
- сложные роли и права;
- автоматические уведомления;
- сложная аналитика;
- корпоративная ERP-логика;
- отдельные режимы для будущих клиентов.

## Минимальные правила
- сайт должен работать локально на `localhost`;
- данные не должны теряться при обычной работе;
- любое важное изменение должно попадать в историю;
- интерфейс должен быть простым и понятным;
- карточка объекта остаётся главным экраном работы.

## Критерий успеха
Первая версия полезна, если пользователь может открыть объект, понять его состояние, изменить данные, сохранить историю и не потерять контекст.
