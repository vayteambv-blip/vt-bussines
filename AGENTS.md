# Инструкции Codex для проекта

## Главные правила
- Всегда отвечай пользователю на русском языке, если он явно не попросил другой язык.
- Все Markdown-файлы, спецификации и пользовательские тексты интерфейса должны быть на русском языке.
- Технические имена файлов, папок, команд, переменных и компонентов можно писать латиницей, если это нужно для совместимости инструментов.
- Перед изменением файлов сначала прочитай релевантные текущие файлы.
- Не перезаписывай пользовательские правки без явной просьбы.

## Суть проекта
- Проект нужен для управления строительными объектами в Бельгии.
- Первая версия должна работать локально на `localhost`.
- На старте это фронтенд-проект без бэкенда, онлайн-деплоя и авторизации.
- В проекте обязательна история: каждое обновление объекта, комментарий, анализ и изменение состояния должны сохраняться.
- Пользователь хочет использовать локальный Codex как ИИ-помощника для маленьких анализов, комментариев и анализа строительных проектов.

## Минимальная структура
- `spec/01-бизнес-контекст.md` - главный спек проекта.
- `spec/02-функциональная-карта.md` - следующий спек после бизнес-контекста.

## Рабочий процесс
1. Сначала работай только с `spec/01-бизнес-контекст.md`.
2. Когда бизнес-контекст становится понятным, переходи к `spec/02-функциональная-карта.md`.
3. Только после согласования бизнес-контекста и функциональной карты можно добавлять технический план и код.
4. Если в спеке нет ответа, задавай вопрос, а не придумывай молча.
5. Если есть противоречие, останавливайся и проси уточнение.

## Разработка через спецификации
- Порядок всегда такой: спецификация -> план -> код -> проверка.
- Нельзя начинать реализацию функции, если нет понятного описания в спеке.
- Нельзя выбирать техническое решение, если бизнес-сценарий ещё не описан.

## OpenAI и Codex
- Если нужно работать с OpenAI API, ChatGPT Apps SDK, Codex или их настройкой, сначала используй официальную документацию OpenAI.
- Если доступен OpenAI developer documentation MCP server, используй его без отдельной просьбы пользователя.
- Не добавляй OpenAI API, ключи, бэкенд или онлайн-интеграции без отдельного подтверждения пользователя.
